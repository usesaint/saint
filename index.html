documentation

simple guides

start here

overview

saint is a minimal server bot for setup, moderation, custom messages, voice rooms, music, last.fm, search, counters, and automations. most systems are managed through small menus so staff do not have to memorize every subcommand.

main prefix

most saint commands use ?. last.fm uses period commands like .fm so music stats stay separate.

server based

settings are meant to stay inside the server where they are changed. one server should not inherit another server’s setup.

menus

commands like ?welcome, ?boost, ?starboard, and ?autorole open managers with add, remove, preview, and reset options.

custom text

message strings support variables and small options, but your custom writing should stay exactly how you set it.

?help?commands?setup?info
first setup

getting started

after inviting saint, run the setup command in the server you want to configure. it creates the basic pieces saint needs, then you can adjust the systems you actually use.

1. run setup

?setup creates the core setup such as logs and moderation basics where possible.

2. check logs

use ?logs if you want the current channel to receive moderation, invite, and system events.

3. open help

?help shows the cleaner in-server help. ?commands shows the full directory.

4. configure features

open each manager only when you need it: welcome, boost, birthday, vanity, autoroles, counters, starboard, and more.

tip: if something does not work, check saint’s role position first. discord blocks role and moderation actions when the bot role is too low.

access

permissions

saint needs different permissions depending on the feature. you do not have to give every permission forever, but setup and moderation work best when saint’s role is high enough.

moderation

manage messages, moderate members, kick, ban, view audit log, and manage roles for jail or mute roles.

messages

send messages, embed links, attach files, read message history, mention roles if you want pingable templates.

roles

manage roles, with saint’s role above every role it needs to add, remove, create, or update.

voice + music

connect, speak, move members, manage channels, and view channels for join-to-create rooms.

?setup?perms?roleinfo
custom text

message builder

custom message systems use one message string. you can write the full message, include variables, choose ping behavior, and preview or test from the manager.

variables

use variables such as user mention, server name, member count, boost count, role mention, and channel mention where supported.

ping option

$ping on or $ping off controls whether a message should actually ping.

small text

$smalltext on or $smalltext off should apply only to supported bot styling, not force your custom message to gray out.

preview + test

manager popups can preview or test before the message is used publicly.

full string example

?boostmsg $channel #boosts $channel2 #general $ping on $smalltext off $color #0c0c0c $thumbnail {user.avatar} $image {guild.icon} $footericon {user.avatar} $footer "blessed by your grace" $description "saint / saynt / n. may your generosity be returned to you, {user.mention}. v. i. blessings: {guild.boost_count}"

?variables?welcome?boost?warn settings
server messages

welcome + boost

welcome and boost are managed from their main commands. choose channels, write the message, preview it, test it, and reset when you want to return to the default.

welcome

?welcome opens the welcome manager for join messages, channel selection, and previews.

boost

?boost opens the boost manager for server boost thank-you messages and boost count variables.

two channels

systems that support channel selection can send to more than one chosen channel when configured.

defaults

reset returns the system to saint’s default message without affecting other servers.

?welcome?boost?welcome reset?boost reset
announcements

birthday + vanity

birthday and vanity systems are built for simple announcements. birthday tracks saved birthdays, while vanity rewards members for using the server vanity text.

birthday

?birthday shows or manages birthday info. setup options live under birthday settings.

vanity

?vanity manages vanity text, rep counts, roles, channels, and announcement messages.

ping control

include $ping on/off in supported message strings when you want to control mentions.

view first

use preview or test in the manager before letting the saved message run automatically.

?birthday?birthday settings?vanity?vanity reset
staff tools

moderation

moderation commands are for fast cleanup and staff action. saint supports warnings, purge, jail, mute, image mute, reaction mute, kick, ban, and giveaways.

warnings

?warn warns a member. ?warn settings changes how warning messages look.

purge

?purge 10 deletes recent messages. ?purge @user 10 targets one user’s recent messages.

jail + mute

jail limits a member to the jail setup. mute/timeouts limit normal chat access.

media limits

image mute and reaction mute are for users who should only lose one type of access.

?warn?purge?jail?mute?ban
protection

anti raid

anti raid watches for fast spam patterns, invite floods, webhook abuse, and sudden suspicious activity. it should protect the server without punishing one normal invite link.

raid patterns

saint looks for repeated behavior: bursts of joins, repeated links, webhook spam, and message floods.

lockdown

lockdown temporarily limits channels when staff need a fast stop button during an active raid.

webhooks

if webhook spam appears, saint can help remove the spam and surface where it came from.

not too strict

the goal is to catch spam waves, not punish a member for one normal invite.

?antiraid?antiraid lockdown?antiraid unlock

automatic systems

automation

automation systems let saint respond, react, post scheduled messages, keep sticky messages at the bottom, update counters, and send starboard posts.

auto response

respond to trigger text with normal text, embeds, role mentions, or other supported options.

auto react

react to matching messages with one or more emojis.

auto + sticky messages

auto messages post on a schedule. sticky messages repost so important info stays visible.

counters

counter channels update numbers for members, humans, bots, roles, or other supported totals.

?autoresponse?autoreact?automessages?sticky?counter?starboard
voice

voice rooms

join-to-create makes temporary voice rooms. members can manage their own room with the voice menu while staff keep logs and server-wide stats.

create rooms

room controls

?vc opens controls for rename, reset, lock, unlock, hide, show, drag, permit, reject, and role access.

voice stats

stats should count time in this server only. logs show sessions; stats show totals.

reset time

?resetvoicetime returns voice totals to zero for the server.

?j2c?vc?stats?resetvoicetime
player

music

music supports search, links, playback controls, queues, playlists, liked songs, volume, and simple now-playing embeds.

play

controls

pause/play, skip, previous, queue, volume, and now playing keep playback simple.

library

liked songs and playlists are user-specific, so each member manages their own saved music.

menu

the music menu groups volume, filters, playlist selection, liked songs, and search where possible.

?play?pause?queue?volume?playlist?like
music stats

last.fm

last.fm commands use period commands. connect your account, show now playing, view recent tracks, and refresh when the cached track feels out of sync.

connect

.login saves your last.fm username for saint’s last.fm commands.

now playing

.fm shows your current track. .sync refreshes if it looks stale.

history

.recent shows recent tracks. charts and collages summarize listening.

links

music link commands can create links for apple music, spotify, youtube, or genius when supported.

.login.fm.sync.recent.chart
media

search + media

search tools help members find images, anime, youtube videos, colors, avatars, banners, definitions, and social profiles without leaving chat.

image search

?image returns multiple image pages and should avoid broken preview links where possible.

anime search

?anime uses anilist-style anime results instead of a general web search.

youtube

?youtube searches videos and returns playable results.

profiles

social commands can save a username per user, then show the saved profile when run without a username.

?image?anime?youtube?color?avatar?roblox
defaults

resets

reset commands return a feature to its default state. resets should affect only the server where the command is used.

full reset

?bot reset confirm resets saint’s server settings when you want to rebuild from default.

message resets

welcome, boost, birthday, vanity, warning, and bump systems can return to default messages.

system clears

autoroles, bump reminders, snipes, quote channels, and join-to-create can be cleared.

safe use

use resets when a setup feels messy or when a new server should start clean.

?bot reset confirm?welcome reset?boost reset?autorole clear?clearsnipe
fixes

troubleshooting

most issues come from permissions, role position, missing channels, old deleted channel IDs, or a feature being configured in the wrong server.

command does nothing

check that saint can view the channel, send messages, embed links, and read message history.

roles fail

move saint’s role above the role it is trying to manage, then try again.

wrong server settings

settings should be saved by server. if something appears copied from another server, reset and re-save it in the current server.

deleted channel error

if logs mention an unknown channel, re-run the setup or reselect the channel for that feature.

if the terminal shows a repeated error, copy the full error and the command you ran before it. that makes the fix much faster.

data

privacy

saint stores only what it needs to run enabled features, such as server settings, saved usernames, message templates, role setups, counters, and user-specific music or last.fm settings.

server settings

configuration is stored per server so one server’s setup should not affect another.

user settings

saved social usernames, last.fm usernames, liked songs, playlists, and AFK data may be stored for convenience.

moderation data

warnings, jail records, and logs may be stored when those systems are enabled.

support

use the support page if you need help, want something removed, or need a data question answered.

privacy policy

june 27, 2026

saint stores only what is needed for its features and settings.

information saint may store

  • server, channel, role, and user ids.
  • server settings and custom message setup.
  • moderation records such as warnings or jail/mute settings.
  • feature data like birthdays, afk, invites, voice time, last.fm, socials, and playlists.
  • message content only when a feature needs it, such as sticky messages, snipes, logs, or auto responses.

how data is used

data is used to run features, remember settings, and keep each server separate.

data sharing

saint does not sell data. some commands may use outside services for music, search, ai edits, translation, or last.fm.
